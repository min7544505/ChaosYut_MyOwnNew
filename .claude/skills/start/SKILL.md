# Start Session Skill

세션 시작 시 이전 진행기록을 읽고 학습 컨텍스트를 복원합니다.

## 역할
- `session-logger` 에이전트와 대칭 구조
- 종료 시 기록 → 시작 시 복원

---

## 실행 단계

### 1. 진행기록 파일 목록 확인
```
E:\UnrealProject\ChaosYut_New\진행기록\*.md
```

### 2. 최신 2-3개 파일 읽기
- 파일명 형식: `YYMMDD_N회차_진행상황.md`
- 날짜 기준 내림차순으로 정렬하여 최신 파일 선택

### 3. 각 세션 요약 출력
각 진행기록에서 다음 정보 추출:
- 세션 날짜/회차
- **완료한 작업** (오늘 한 일 섹션)
- **결정된 사항** (간략히 핵심만)

### 4. 가장 최근 세션의 "다음에 이어갈 것" 표시
- 체크리스트 형태로 표시
- 우선순위가 있다면 표시

### 5. 사용자에게 질문
어떤 작업부터 시작할지 선택하도록 안내

---

## 출력 형식

```markdown
## 이전 세션 요약

### [N회차] YYYY-MM-DD
**완료한 작업:**
- 작업 1
- 작업 2

**주요 결정:**
- 결정 사항 요약

---

### [N-1회차] YYYY-MM-DD
**완료한 작업:**
- 작업 1

---

## 다음에 이어갈 것 (최신 세션 기준)

- [ ] 할 일 1
- [ ] 할 일 2
- [ ] 할 일 3

---

오늘은 어떤 작업부터 시작할까요?
1. 위 목록에서 선택
2. 새로운 작업 시작
3. 현재 상황 설명 요청
```

## 실행 방법

```
/start           - 기본 실행 (최신 2-3개 요약)
/start all       - 전체 진행기록 요약
/start [N]       - 최신 N개 진행기록 요약
```

## 중요: 추가 구현 문서 처리

### 워크플로우 배경
사용자는 Claude Code 사용량 한계로 인해:
1. Claude Code에서 플랜/설계를 받음
2. 별도 사이트(웹 Claude 등)에서 추가 학습 및 구현
3. 구현 결과를 `_추가구현.md` 등의 문서로 기록

### 파일 패턴
```
260117_2회차_진행상황.md          ← Claude Code 세션 기록
260117_2회차_진행상황_추가구현.md  ← 사용자가 별도로 학습/구현한 내용
```

### 필수 참조 규칙
**추가구현 문서가 있으면 반드시 함께 읽어야 함!**
- 같은 회차의 `_추가구현`, `_추가`, `_구현` 등 접미사 파일 모두 확인
- 이 문서들에 실제 구현된 코드/로직이 상세히 기록되어 있음
- 메인 진행기록만 보면 컨텍스트가 불완전함

### 요약 시 통합
```markdown
### [2회차] 2026-01-17
**Claude Code 세션:**
- 플랜 작성 완료

**추가 구현 (별도 학습):**
- SpawnRoadByType 함수 구현
- InitializeBoard 함수 구현
- Map 자료구조 학습
```
